# 系统化调试

按 4 步流程排查 Bug，**找到根因才能修，不猜不蒙。**

## 当前改动

！`git log --oneline -5`

！`git diff HEAD`

## 四步流程

### 1. 复现
- 操作步骤是什么？
- 期望结果 vs 实际结果？
- 稳定复现还是偶发？

### 2. 缩小范围
- 最近改了哪些文件？
- 用二分法：注释掉最近改动，Bug 还在吗？
- 日志里有什么线索？

参照 `scripts/check_logs.sh` 查看日志。

### 3. 分析根因
找到问题的根本原因，不是表面现象。

### 4. 修复验证
- 改代码
- 确认 Bug 修复
- 检查没引入新问题

## Red Flags

如果出现以下想法，说明在**猜**，不是**查**：
- "先快速修一下，之后再查原因"
- "试试改 X 看看行不行"
- "一次性改多个地方"
- "跳过测试，手动验证"
- 已经尝试 3 次以上还没修好
